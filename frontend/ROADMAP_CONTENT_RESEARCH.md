## 1. Frontend Development

**Template**: `learn-frontend` | Icon: `🎨` | Color: `cyan` | Duration: 30 days | Difficulty: beginner | Category: Coding & Tech

**Description**: From HTML basics to deploying a React app — build real UIs every single day.

### Day-by-Day Roadmap

| Days | Phase | Daily Tasks |
|------|-------|-------------|
| 1-2 | HTML Foundations | Day 1: Semantic HTML — headings, paragraphs, lists, links, images. Build a personal bio page. Day 2: Forms, tables, media embeds. Build a survey form. |
| 3-5 | CSS Fundamentals | Day 3: Selectors, box model, colors, fonts. Style the bio page. Day 4: Flexbox — align, justify, wrap. Build a navigation bar + card grid. Day 5: CSS Grid — template areas, responsive layouts. Build a photo gallery. |
| 6-7 | Responsive Design | Day 6: Media queries, mobile-first design. Make all previous projects responsive. Day 7: CSS variables, transitions, animations. Add hover effects and a dark mode toggle. |
| 8-10 | JavaScript Core | Day 8: Variables, types, operators, conditionals, loops. Day 9: Functions, scope, closures, arrays, objects. Day 10: DOM manipulation — selectors, event listeners, dynamic content. Build an interactive to-do list. |
| 11-13 | JS Intermediate | Day 11: ES6+ — arrow functions, destructuring, spread/rest, template literals. Day 12: Async JS — callbacks, promises, async/await, fetch API. Build a weather app. Day 13: Error handling, localStorage, JSON. Persist the to-do list. |
| 14-15 | Tooling | Day 14: npm, package.json, scripts. Set up a project with Vite. Day 15: ESLint, Prettier, Git basics for frontend devs. |
| 16-18 | React Intro | Day 16: JSX, components, props. Build a profile card component. Day 17: State with useState, event handling. Build a counter + toggle. Day 18: Lists, conditional rendering, keys. Build a filterable product list. |
| 19-21 | React Intermediate | Day 19: useEffect, data fetching, loading states. Fetch from a public API. Day 20: Forms in React — controlled components, validation. Day 21: React Router — pages, navigation, URL params. Multi-page app. |
| 22-24 | State & Patterns | Day 22: Lifting state up, component composition. Day 23: Context API for global state (theme, auth). Day 24: Custom hooks — useLocalStorage, useFetch, useDebounce. |
| 25-27 | Styling in React | Day 25: Tailwind CSS setup + utility-first approach. Day 26: Component library — build Button, Card, Modal. Day 27: Animations with Framer Motion. |
| 28-29 | Testing & Performance | Day 28: Unit testing with Vitest + React Testing Library. Day 29: Performance — React.memo, useMemo, useCallback, lazy loading. |
| 30 | Deploy | Day 30: Build + deploy to Vercel. Custom domain, environment variables, CI/CD preview. |

**Lesson 5: React — Thinking in Components**

**Lesson 6: React Hooks Deep Dive**
- Steps: (1) useState — managing local state (2) useEffect — side effects and lifecycle (3) useContext — global state without prop drilling (4) useRef — accessing DOM and persisting values (5) Custom hooks — extracting reusable logic

**Lesson 7: Routing & Data Fetching**
- Steps: (1) Client-side routing with React Router (2) Dynamic routes and URL parameters (3) Data fetching patterns — useEffect vs libraries (4) Loading, error, and empty states (5) Caching and revalidation basics

**Lesson 8: Deployment & Performance**

| 8-10 | Databases | Day 8: PostgreSQL setup, SQL basics, CREATE/INSERT/SELECT. Day 9: JOINs, indexes, migrations. Connect Express to PostgreSQL. Day 10: ORMs — Prisma or Drizzle. Rewrite the notes API with an ORM. |

| 20-21 | Testing | Day 20: Unit testing with Jest — services, utilities. Day 21: Integration testing — supertest for API endpoints, test database. |
| 22-24 | Security | Day 22: SQL injection, XSS, CSRF — understand and prevent. Day 23: HTTPS, Helmet.js, input sanitization. Day 24: Environment variables, secrets management, .env best practices. |
| 25-27 | Architecture | Day 25: Project structure — modules, layers, separation of concerns. Day 26: Error handling patterns, logging with Pino/Winston. Day 27: TypeScript for backend — types, interfaces, Zod validation. |

| 30 | Capstone | Day 30: Build and deploy a complete API — auth, CRUD, file uploads, real-time, documented with Swagger. |

**Lesson 1: How the Web Works (for Backend Devs)**
- Steps: (1) Client-server model (analogy: restaurant — client orders, server kitchen prepares) (2) HTTP protocol — methods, headers, status codes (3) DNS resolution — how domains find servers (4) Request lifecycle — from browser to server and back (5) REST principles
- Visual: HTTP request/response flow diagram, status code reference table

**Lesson 2: Node.js — JavaScript on the Server**
- Steps: (1) V8 engine + libuv = Node.js (analogy: Node is a chef who takes orders non-stop but delegates cooking to helpers) (2) Event loop deep dive (3) CommonJS vs ES Modules (4) npm ecosystem — packages, scripts, lock files (5) Building a CLI tool

**Lesson 3: Express.js & Middleware**
- Steps: (1) What is middleware? (analogy: airport security checkpoints) (2) Routing — matching URLs to handlers (3) Request body parsing — JSON, form data (4) Error handling — centralized error middleware (5) CORS — why browsers block cross-origin requests
- Visual: Middleware pipeline flow diagram

**Lesson 4: Database Integration**
- Steps: (1) Choosing a database (comparison table: PostgreSQL vs MySQL vs MongoDB) (2) Connection pooling (analogy: phone lines to a call center) (3) Migrations — version control for your schema (4) ORM vs raw SQL trade-offs (5) N+1 query problem and how to avoid it
- Visual: Connection pool diagram, ORM vs SQL comparison

**Lesson 5: Authentication & Authorization**
- Steps: (1) Hashing vs encryption (analogy: hashing is a meat grinder — you can't un-grind) (2) JWT anatomy — header.payload.signature (3) Session-based vs token-based auth (4) OAuth 2.0 flow — "Login with Google" (5) Role-based access control

**Lesson 6: Caching & Performance**
- Steps: (1) Why cache? (analogy: keeping a sticky note on your desk vs walking to the filing cabinet) (2) Cache-aside pattern (3) Redis data structures (4) Cache invalidation strategies (5) HTTP caching headers
- Visual: Cache hit/miss flow, TTL timeline diagram

**Lesson 7: Security Essentials**
- Steps: (1) SQL injection — how it works and parameterized queries (2) XSS — stored, reflected, DOM-based (3) CSRF — why tokens matter (4) Rate limiting — protect against brute force (5) OWASP Top 10 overview

**Lesson 8: Deployment & DevOps Basics**
- Steps: (1) Docker — containerize your app (analogy: shipping containers standardize transport) (2) Environment management — dev, staging, prod (3) Reverse proxy with Nginx (4) Process management with PM2 (5) CI/CD basics — automated testing and deployment

## Cross-Cutting Engagement Principles

These principles should apply to ALL content areas:

1. **Visual-first**: Every concept should have a diagram, comparison table, or info card. No walls of text.
2. **Analogy-driven**: Every abstract concept gets a concrete real-world analogy.
3. **Progressive disclosure**: Start simple, add complexity. Never dump everything at once.
4. **Daily deliverables**: Every day in a roadmap produces something tangible — a file, a screenshot, a recording, a completed exercise.
5. **Social proof**: Show participant counts, completion rates, and group activity.
6. **Micro-celebrations**: Confetti animation on day completion, streak badges, milestone cards.
7. **Spaced repetition**: Review days built into every roadmap (Day 7, 14, 21, etc.).
8. **Real-world context**: Use real companies, real tools, real scenarios — not abstract toy examples.
